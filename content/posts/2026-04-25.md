### 1. Are Current AI Virtual Cell Models Useful for Scientific Discovery?
- Original Title: Are Current AI Virtual Cell Models Useful for Scientific Discovery?
- Keywords: virtual cell, perturbation model, benchmark, hit discovery
- 简要描述: 这篇工作质疑当前虚拟细胞模型的评估方式，认为只看表达预测误差并不能回答它们是否真的有助于科学发现。
- 研究主题: 面向 hit discovery 的虚拟细胞评测框架
- 背景: 许多 perturbation 模型在 benchmark 上看起来表现不错，但这些分数与下游发现任务之间的关系并不清楚。
- 方法: 作者提出 PerturbHD，把评测目标直接对齐到 AI-enabled hit discovery，而不是停留在静态预测拟合上。
- 结果: 论文用这一框架展示了按发现价值评测模型与按常规预测指标评测模型之间可能存在明显偏差。
- 意义: 这提醒人们，虚拟细胞研究的关键不只是“拟合得像不像”，而是“能不能支持更有效的实验筛选与发现决策”。
- 原文链接: https://www.biorxiv.org/content/10.64898/2026.04.23.719015v1

### 2. Housekeeping Gene Expression Normalization in Transcriptomics Mitigates Data Leakage in Machine Learning Models
- Original Title: Housekeeping Gene Expression Normalization in Transcriptomics Mitigates Data Leakage in Machine Learning Models
- Keywords: normalization, data leakage, housekeeping genes, transcriptomics
- 简要描述: 这项工作把一个常被忽视的问题挑明了: 转录组归一化本身也可能把机器学习结果“洗得过于好看”。
- 研究主题: RNA-seq 归一化中的泄漏控制与稳健建模
- 背景: 常用 housekeeping genes 在不同临床条件下并不总是稳定，如果拿它们做归一化，模型可能无意中利用了泄漏信号。
- 方法: 作者在肾移植公开数据上，用变异系数、稳定性和 Gini 系数筛选更稳定的 housekeeping genes，并与常见归一化方法比较。
- 结果: 识别出的候选基因在泛化和稳健性上优于 median ratio normalization 与 TMM 等基线。
- 意义: 这说明 transcriptomics 里的“前处理细节”并不是次要工程问题，而会直接决定下游模型是否可信。
- 原文链接: https://www.biorxiv.org/content/10.64898/2026.04.24.720637v1

### 11. Tool Attention Is All You Need: Dynamic Tool Gating and Lazy Schema Loading for Eliminating the MCP/Tools Tax in Scalable Agentic Workflows
- Original Title: Tool Attention Is All You Need: Dynamic Tool Gating and Lazy Schema Loading for Eliminating the MCP/Tools Tax in Scalable Agentic Workflows
- Keywords: MCP, tool gating, schema loading, context efficiency
- 简要描述: 论文关注一个非常工程化但实际很痛的问题: 工具 schema 本身就可能把 agent 的上下文预算吃掉。
- 研究主题: agent 工具上下文的动态压缩与选择
- 背景: 在多 server、多工具部署里，eager schema injection 会带来巨大的 token 与 cache 开销。
- 方法: 作者提出 Tool Attention，用 intent overlap、状态约束和 lazy schema loading 只把 top-k 工具完整展开。
- 结果: 在模拟 benchmark 中，该方法大幅减少每轮工具上下文 token，并把有效上下文利用率显著抬高。
- 意义: 它强调 scalable agent 的瓶颈未必是模型上下文窗口，而是协议层怎样管理工具信息。
- 原文链接: https://arxiv.org/abs/2604.21816
